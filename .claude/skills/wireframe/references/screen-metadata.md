# Screen Metadata — 화면 헤더 메타데이터 및 ID 체계

> 모든 화면설계서의 화면ID 부여, 버전 관리, 헤더 필드 작성에 이 문서를 기준으로 적용.

---

## 1. 화면ID 체계

### 1.1. 형식

```
[플랫폼 접두어][화면그룹번호(2자리)][화면순번(4자리)]
```

| 플랫폼 | 접두어 | 예시 |
|---|---|---|
| PC (웹 데스크탑) | `PC` | PC200001 |
| Mobile (모바일 웹) | `M` | M200001 |
| Tablet (필요 시 추가) | `T` | T200001 |

### 1.2. 화면그룹 번호 (기본값 — 프로젝트별 조정 가능)

| 그룹번호 | 그룹명 | 해당 화면 예시 |
|---|---|---|
| 10 | 공통 | GNB, 푸터, 모달, 토스트 |
| 20 | 홈 | 메인 페이지, 랜딩 |
| 30 | 프로젝트 | 프로젝트 목록, 상세 |
| 40 | 서비스 | 서비스 목록, 상세 |
| 50 | 스페이스 | 스페이스 목록, 상세 |
| 60 | 마이페이지 | 프로필, 설정, 내 활동 |
| 70 | 등록하기 | 프로젝트 등록, 서비스 등록 |
| 80 | 인증 | 로그인, 회원가입, 비밀번호 찾기 |
| 90 | 관리자 | 어드민 전용 화면 |

프로젝트 특성에 따라 그룹 추가 및 번호 재정의 가능.
변경 시 IA 문서에 화면그룹 테이블을 반드시 업데이트.

### 1.3. 화면순번 규칙

- 4자리 숫자: `0001`부터 시작
- 동일 그룹 내 화면이 추가될 때마다 순번 증가: `0001`, `0002`, `0003`...
- 삭제된 화면의 순번은 결번으로 처리 (재사용 금지)
- 팝업/모달은 부모 화면 순번에 알파벳 추가: `PC200001A` (홈의 팝업)

### 1.4. PC/Mobile 동일 화면 처리

동일 화면이라도 플랫폼별로 별도 화면ID 부여. 같은 순번 사용 가능:

| 화면ID | 화면명 | 플랫폼 |
|---|---|---|
| PC200001 | 홈 | PC |
| M200001 | 홈 | Mobile |

## 2. 화면 헤더 필드

모든 화면설계서 블록 최상단에 아래 헤더 포함:

```
화면 경로: /path/to/screen
화면 ID: PC200001
버전: v0.1.0
화면명: [화면의 공식 명칭]
작성자: [담당자명]
작성일: YYYY.MM.DD(요일)
```

### 2.1. 각 필드 작성 규칙

**화면 경로**
- 실제 URL 경로 기준: `/`, `/se/{slug}`, `/my/profile`
- 동적 파라미터는 `{slug}`, `{id}` 형식으로 표기
- 쿼리 파라미터는 생략 가능 (중요한 경우만 표기: `/search?q={keyword}`)

**화면 ID**
- 위 1.1~1.3 규칙 준수
- 헤더의 화면ID와 와이어프레임 내 참조 표기 일치 필수

**버전**
- 아래 3. 버전 규칙 참조

**화면명**
- IA 사이트맵의 화면명과 동일하게 사용
- 임의로 바꾸지 않음 — 변경 필요 시 IA 문서 먼저 업데이트

**작성자**
- 화면을 실제 작성한 담당자명 (팀명이 아닌 개인 이름)

**작성일**
- 포맷: `2026.05.17(일)` 형식
- 최초 작성일 기준 — 수정 시 버전만 올리고 작성일은 유지

---

## 3. 버전 규칙

### 3.1. 버전 형식

```
v[MAJOR].[MINOR].[PATCH]
```

| 자리 | 의미 | 증가 시점 |
|---|---|---|
| MAJOR | 단계 | Lo-fi(0) → Hi-fi(1) → 운영 반영(2) |
| MINOR | 마이너 수정 | 어노테이션 업데이트, 영역 추가/삭제 |
| PATCH | 패치 | 오타 수정, 링크 수정 |

### 3.2. 단계별 버전 범위

| 버전 범위 | 단계 | 설명 |
|---|---|---|
| `v0.x.x` | Lo-fi 와이어프레임 | 현재 스킬의 산출물 |
| `v1.x.x` | Hi-fi 디자인 완료 | 브랜드 컬러·폰트·실제 이미지 적용 후 |
| `v2.x.x` | 개발 반영 완료 | QA 통과 후 운영 기준 버전 |

### 3.3. 버전 증가 예시

```
v0.1.0 → v0.1.1  : 어노테이션 오타 수정
v0.1.1 → v0.2.0  : GNB 영역 어노테이션 대폭 수정
v0.2.0 → v1.0.0  : Hi-fi 디자인 완료 단계 진입
v1.0.0 → v1.1.0  : 클라이언트 피드백 반영 (버튼 텍스트 변경)
```

### 3.4. 버전 혼재 금지

- 동일 화면의 여러 버전(예: `홈_v0.1.0`, `홈_v0.1.1`)이 같은 문서에 공존 금지
- 현행 버전 하나만 유지
- 이전 버전 내용은 문서 하단 Changelog 섹션에 기록

## 4. Changelog 형식

각 화면설계서 문서 하단에 Changelog 섹션 포함:

```markdown
## Changelog

| 버전 | 날짜 | 작성자 | 변경 내용 |
|---|---|---|---|
| v0.1.0 | 2026.05.17 | 홍길동 | 최초 작성 |
| v0.1.1 | 2026.05.20 | 홍길동 | 03 섹션 인터랙션 오타 수정 |
| v0.2.0 | 2026.05.25 | 홍길동 | GNB 영역 비로그인 상태 추가 |
```

## 5. 전체 화면 인벤토리 테이블 예시

화면설계서 문서 맨 앞에 전체 화면 목록 테이블 포함:

| 화면ID | 화면명 | 경로 | 기능ID | 플랫폼 | 버전 | 상태 |
|---|---|---|---|---|---|---|
| PC100001 | 공통 GNB | (공통) | F001 | PC | v0.1.0 | 작성 완료 |
| M100001 | 공통 GNB | (공통) | F001 | Mobile | v0.1.0 | 작성 완료 |
| PC200001 | 홈 | / | F002 | PC | v0.1.0 | 작성 완료 |
| M200001 | 홈 | / | F002 | Mobile | v0.1.0 | 작성 중 |
| PC300001 | 프로젝트 목록 | /projects | F010 | PC | v0.1.0 | 미작성 |

**상태 값:**
- `미작성` — 화면 인벤토리에만 등재, 설계서 아직 없음
- `작성 중` — 초안 작성 진행 중
- `작성 완료` — Lo-fi 완성
- `클라이언트 검토 중` — 피드백 대기
- `승인` — 클라이언트 최종 승인 완료
