# PRD Template

## 문서 구조

### 1. 제품 개요
- **제품명**: 
- **한 줄 설명**: (타겟 유저)가 (핵심 문제)를 해결하기 위한 (솔루션)
- **목표**: 이 제품/기능으로 달성하려는 비즈니스 목표
- **비목표**: 이번 범위에서 의도적으로 제외하는 것

### 2. 타겟 유저
- **페르소나**: 이름·역할·Pain Point
- **주요 시나리오**: 유저가 이 제품을 쓰게 되는 상황

### 3. 기능 목록

| 기능명 | 우선순위 | 설명 | 완료 기준 |
|---|---|---|---|
| | P0 | | |
| | P1 | | |
| | P2 | | |

우선순위 기준:
- **P0**: MVP 필수. 없으면 제품 성립 안 됨
- **P1**: 1차 출시 후 추가
- **P2**: 장기 로드맵

### 4. 기능 상세 (P0 중심)

각 기능마다:
```
#### [기능명] — P0
완료 기준: [공수·범위·제약 한 줄]

**동작 설명**
- 정상 흐름: ...
- 엣지 케이스: ...
- 에러 처리: ...

**비기능 요구사항**
- 응답 시간: [확인 필요]
- 동시 사용자: [확인 필요]
```

### 5. 비기능 요구사항
- 성능: [확인 필요]
- 보안: 인증 방식, 데이터 보호
- 접근성: [확인 필요]

### 6. 개방 질문
PRD 작성 중 결정이 필요한 항목 목록 (`[결정 필요]` 마커와 연동)

### 7. 추진 일정 (추정)
| 단계 | 내용 | 기간 |
|---|---|---|
| 설계 | | [확인 필요] |
| 개발 | | [확인 필요] |
| 테스트 | | [확인 필요] |

## 작성 원칙
- 수치 근거 없는 일정·공수 임의 기재 금지 → `[확인 필요]` 표기
- "혁신적", "차별화된" 같은 빈 수식어 금지 — 구체적 기능으로 대체
- 각 기능 작성 전 완료 기준 한 줄 명시 후 시작
