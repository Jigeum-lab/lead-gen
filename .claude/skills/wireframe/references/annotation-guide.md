# Annotation Guide — Description 패널 작성 가이드

> Phase On 화면설계서 Description 패널 분석 기반. 동일 품질 기준을 모든 클라이언트 화면설계서에 적용.

---

## 1. Description 패널 구조

```
01 [영역명]
• 표출 데이터: [어떤 데이터가 표시되는가 — 구체적으로]
• 인터랙션: [트리거 → 결과] 형식
• 상태:
  - 정상: [정상 표현]
  - 로딩: [로딩 표현]
  - 에러/빈 상태: [에러 또는 데이터 없는 경우]
• 기능 참조: F001

02 [영역명]
• 표출 데이터: ...
• 인터랙션: ...
• 상태:
  - 정상: ...
  - 로딩: ...
  - 에러/빈 상태: ...
• 기능 참조: F002
```

번호(01, 02, 03...)는 ASCII 와이어프레임의 번호 태그(①②③)와 1:1 매핑.
번호 누락 금지, 순서 불일치 금지.

## 2. Phase On 홈 화면 Description 실제 예시

Phase On 화면설계서에서 관찰한 홈 화면 Description 패널 구조:

**01 GNB (공통 헤더)**
- 헤더 메뉴 표시 여부 — 전 페이지 공통 적용
- 로그인 상태: `[등록하기]` + `[마이페이지]` 버튼 표시
- 비로그인 상태: `[로그인]` 버튼 표시, `[등록하기]` 버튼 비활성
- 표출 데이터: 로고, 메뉴(홈/프로젝트/서비스/스페이스), 우측 액션 버튼

**02 히어로 배너**
- 이미지: placeholder 1200×500px (실제 에셋은 Hi-fi 단계에서 삽입)
- 캐러셀 여부: 기획 미확정 `(확인 필요)`
- 인터랙션: 좌우 화살표 클릭 → 다음/이전 배너로 전환, 자동 슬라이드 5초 간격 `(확인 필요)`

**03 프로젝트/아티클 리스트**
- 표출 데이터: 최신 프로젝트 카드 N개 (정렬 기준: 등록일 내림차순)
- 카드 구성: 썸네일 + 제목 + 카테고리 태그 + 작성자 + 등록일
- 인터랙션: 카드 클릭 → 프로젝트 상세 페이지(`/se/{slug}`)로 이동
- 상태:
  - 정상: 카드 N개 표시 (N값 기획 확정 필요)
  - 로딩: 카드 영역 스켈레톤 UI 표시
  - 빈 상태: "등록된 프로젝트가 없습니다" 메시지 + 등록 유도 버튼

**04 푸터**
- 표출 데이터: 이용약관 링크, 개인정보처리방침 링크, 지금.lab 링크, 사업자 정보
- 인터랙션: 링크 클릭 → 각 페이지로 이동 (새 탭 여부 미확정 `(확인 필요)`)

## 3. 어노테이션 품질 기준

### 3.1. 표출 데이터 — 구체적으로

| 나쁜 예 (X) | 좋은 예 (O) |
|---|---|
| "버튼이 있다" | "로그인 상태: '등록하기' 버튼 / 비로그인 상태: '로그인' 버튼" |
| "목록이 표시된다" | "최신 프로젝트 카드 6개, 정렬: 등록일 내림차순" |
| "이미지가 있다" | "썸네일 이미지 placeholder 400×300px" |
| "텍스트가 있다" | "서비스 제목(최대 40자), 카테고리 태그(최대 3개)" |

### 3.2. 인터랙션 — 트리거 → 결과 형식

```
[트리거] → [결과]
```

예시:
- `클릭 → 프로젝트 상세 페이지(/se/{slug})로 이동`
- `호버 → 카드 그림자 강조 + 커서 pointer 전환`
- `폼 제출 → 로딩 스피너 표시 → 성공 시 확인 페이지(/complete)로 이동`
- `스크롤 다운 → 다음 페이지 데이터 로드 (무한 스크롤)`

### 3.3. 상태 — 최소 3가지 필수

모든 데이터 영역에 최소 3가지 상태 기술:

| 상태 | 설명 | 예시 |
|---|---|---|
| 정상 | 데이터 정상 로드된 상태 | "카드 6개 그리드 표시" |
| 로딩 | 데이터 요청 중 상태 | "스켈레톤 UI 6개 표시" |
| 에러/빈 상태 | 오류 또는 데이터 없음 | "'프로젝트가 없습니다' + 등록 유도 버튼" |

로그인/비로그인 분기가 있는 영역은 추가로 명시:

```
• 상태:
  - 로그인: '등록하기' 버튼 활성
  - 비로그인: '로그인' 버튼 표시
  - 로딩: 버튼 영역 스켈레톤
  - 에러: 버튼 비활성 + 툴팁 "잠시 후 다시 시도해주세요"
```

### 3.4. 불확실 항목 처리

확정되지 않은 수치·기획 사항은 placeholder로 두지 말고 `(확인 필요)` 태그 사용:

```
• 표출 데이터: 최신 프로젝트 카드 N개 (N값 확인 필요)
• 인터랙션: 자동 슬라이드 간격 5초 (확인 필요)
```

## 4. Figma 어노테이션 삽입 규칙

### 4.1. 위치 규칙

- Description 패널 내용은 Figma 프레임 내 우측 패널에만 삽입
- 프레임 밖 텍스트 노드 생성 금지 — "페이지명 / 적용 기능 / 적용 상세" 등 설명 텍스트를 프레임 외부에 배치하면 안 됨

### 4.2. 삽입 방법

**Figma 주석(Comment) 방식:**
1. Figma에서 해당 프레임 선택
2. `C` 단축키 → 주석 모드 진입
3. 번호 영역(①②③) 위치에 핀 배치
4. Description 패널 해당 섹션 내용 붙여넣기

**Annotation 플러그인 방식 (권장):**
1. Figma 플러그인 > Annotation 또는 Redline 플러그인 설치
2. 영역별 어노테이션 레이어 생성
3. 번호 태그와 Description 패널 내용 연결

### 4.3. 프레임명 규칙

```
[화면ID] 화면명 v버전
예: [PC200001] 홈 v0.1.0
    [M200001] 홈 (Mobile) v0.1.0
```

## 5. 체크리스트

화면설계서 완성 전 아래 항목 전부 확인:

- [ ] 모든 번호 영역(①②③)에 Description 패널 섹션(01, 02, 03) 1:1 매핑 완료
- [ ] Placeholder 텍스트("Description 내용", "내용을 입력하세요") 없음
- [ ] 모든 데이터 영역에 정상/로딩/에러 3가지 상태 기술
- [ ] 인터랙션은 "트리거 → 결과" 형식으로 기술
- [ ] 불확실 항목은 `(확인 필요)` 태그 사용
- [ ] 기능ID(F001 등) 참조 표기 완료
- [ ] 컬러 코드, 실제 이미지 없음 (Lo-fi 준수)
