## 비용·품질 트레이드오프

```
비용 낮음                                         비용 높음
   │                                                   │
Haiku ──────────── Sonnet ──────────────────── Opus
   │                                                   │
 속도 빠름                                       추론 깊음
 반복 작업                                       고위험 판단
 체크리스트                                      아키텍처 결정
```

**규칙**:
- Haiku를 기본값으로 → 품질 부족하면 Sonnet으로 올린다.
- Opus는 판단이 실제로 필요한 곳만 — 리서치 통합, PRD 크리틱, 에스컬레이션. 나머지는 Sonnet.
- Gemini는 Claude가 할 수 없는 것(실시간 웹)만 사용.

## Gemini 호출 패턴

두 가지 경로. 인라인 검증은 Claude의 **WebSearch 툴**을 우선 사용한다(같은 컨텍스트, 별도 인증 불필요). 독립적 2차 그라운딩(다른 모델의 시각)이 필요하면 **Gemini CLI**를 호출한다.

```bash
# Gemini CLI — 단일 클레임 검증 (검증된 호출 형식)
GEMINI_CLI_TRUST_WORKSPACE=true gemini --skip-trust -m gemini-2.5-flash \
  -p "{검증 주제 + 구체적 클레임 + 출처 URL과 ✅/⚠️/❌ 등급으로 답하라는 지시}" \
  > {출력 경로} 2>/dev/null

# 배치 검증 (여러 클레임) → 클레임별 별도 호출.
#   run_in_background로 병렬 실행 권장. GEMINI_API_KEY 환경변수 필요.
```

**Gemini 사용 시점**:
- research.md 수치 검증 (TAM·성장률·경쟁사 MAU 등)
- 경쟁사 최근 6개월 내 동향 확인
- 정부 정책·규제 최신 현황

**Gemini 사용 금지 시점**:
- 문서 작성·편집 (Claude가 담당)
- 내부 데이터·클라이언트 정보 (외부 전송 불가)
